IoT device simulator
Configuration file details

The pyIoTDevSim.yml is the configuration file searched by pyIoTDevSim to sets system, LoRa and projects parameters.
It uses a YAML format and all sections and keys parameters must be defined in lower case mode.
Two main sections are defined:

  • system
  • projects

The system section layout defines common LoRa parameters, LoRa module interface and the debug level used. An example is showed bellow:

system:
debug_level: "DEBUG"
lora_module: "RHF76-052"
serial:
port: "/dev/ttyUSB0"
baudrate: 9600
lora:
base_band: "AU920"
sub_band: 2
class: "A"
rxwin2_freq: 923.3
rxwin2_dr: "DR8"
auth_mode: "LWABP"

The projects section will list the projects enabled to run. An example is showed bellow (in this case Project C is defined but not will be enabled):

projects:
- Project_A
- Project_B
# - Project_C

Each project listed into the projects section must have a section with its name and the following parameters:

Project_A:
sampling_period: 30
ttn:
dev_eui: "0123456789ABCDEF"
app_eui: "0123456789ABCDEF"
dev_addr: "01234567"
nwks_key: "0123456789ABCDEF0123456789ABCDEF"
apps_key: "0123456789ABCDEF0123456789ABCDEF"
app_key: "0123456789ABCDEF0123456789ABCDEF"
lora:
tx_power: 20
uplink_dr: "DR0"
chan0_freq: 917.2
chan0_dr: "DR0"
chan1_freq: 917.9
chan1_dr: "DR0"
adr: "OFF"
repeat: 2
retry: 2
initial_port: 10
sensor_list:
- air_temperature
- air_humidity
- light_intensity
- uv_index
- battery_voltage
air_temperature:
data_type: float32_compressed
min_value: 22.0
max_value: 24.0
air_humidity:
data_type: uint8
min_value: 75
max_value: 80
light_intensity:
data_type: uint16
min_value: 35000
max_value: 38000
uv_index:
data_type: uint8
min_value: 9
max_value: 10
battery_voltage:
data_type: float_uint16
min_value: 4.8
max_value: 5.2

Into subsection sensor_list must be defined the sensor list used by project. Each sensor must have a subsection with its name. This subsection must have data_type, min_value and max_value. Sensor values are randomly generated between values defines into min_value and max_value. The data_type will define the data type and size used to compose LoRa message (all messages are sent using hexadecimal format). The following data types can be used:

  • uint8
    • range: 0 to 255
    • size: 1 byte
  • uint16
    • range: 0 to 65535
    • size: 2 bytes
  • float_uint16
    • range: 0 to 655.35
    • size: 2 bytes
    • limitation: only 2 digits of precision
    • operation: value * 100
  • float32_compressed
    • range: 0 to 655.35
    • size: 3 bytes
    • limitation: byte 0 is the sign (AA = + / AF = -); only 2 digits of precision
    • operation: value * 100
  • float32
    • range: -3.4E+38 to +3.4E+38
    • size: 4 bytes
    • operation: sign bit, 8 bits exponent, 23 bits mantissa
