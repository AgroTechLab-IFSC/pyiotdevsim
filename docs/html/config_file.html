Configuration file details

The pyIoTDevSim.yml is the configuration file searched by pyIoTDevSim to sets system, LoRa and projects parameters.


It uses a YAML format and all sections and keys parameters must be defined in lower case mode.
Two main sections are defined:

  • system
  • projects

The system section layout defines common LoRa parameters, LoRa module interface and the debug level used. An example is showed bellow:

system: // parameters are the same to all projects;
debug_level: "DEBUG" // pyIoTDevSim debug level;
lora_module: "RHF76-052" // LoRa module;
serial: // subsection with serial parameters;
port: "/dev/ttyUSB0" // serial port;
baudrate: 9600 // serial baudrate;
lora: // subsection with LoRa parameters;
base_band: "AU920" // LoRa base band;
sub_band: 2 // LoRa sub band;
class: "A" // LoRa class;
rxwin2_freq: 923.3 // LoRa RX window 2 frequency;
rxwin2_dr: "DR8" // LoRa RX window 2 datarate;
auth_mode: "LWABP" // LoRa authentication mode;

The projects section will list the projects enabled to run. An example is showed bellow (in this case Project C is defined but not will be enabled):

projects:
- Project_A
- Project_B
# - Project_C

Each project listed into the projects section must have a section with its name and the following parameters:

Project_A:
sampling_period: 30 // sampling period (in seconds);
ttn: // subsection with TTN parameters;
dev_eui: "0123456789ABCDEF" // device EUI;
app_eui: "0123456789ABCDEF" // application EUI;
dev_addr: "01234567" // device address;
nwks_key: "0123456789ABCDEF0123456789ABCDEF" // network session key;
apps_key: "0123456789ABCDEF0123456789ABCDEF" // application session key;
app_key: "0123456789ABCDEF0123456789ABCDEF" // application key;
lora: // subsection with LoRa parameters;
tx_power: 20 // transmission power (in dB);
uplink_dr: "DR0" // uplink datarate;
chan0_freq: 917.2 // channel 0 frequency;
chan0_dr: "DR0" // channel 0 datarate;
chan1_freq: 917.9 // channel 1 frequency;
chan1_dr: "DR0" // channel 1 datarate;
adr: "OFF" // Adaptive Data Rate;
repeat: 2 // transmission repeat number;
retry: 2 // transmission retry number;
initial_port: 10 // initial transmission port;
sensor_list: // subsection with list of sensor names;
- air_humidity
- light_intensity
- battery_voltage
- air_temperature
- latitute
- longitude
air_humidity:
data_type: uint8
min_value: 75
max_value: 80
light_intensity:
data_type: uint16
min_value: 35000
max_value: 38000
battery_voltage:
data_type: float_uint16
min_value: 4.82
max_value: 5.28
air_temperature:
data_type: float32_compressed
min_value: -22.65
max_value: 32.48
latitude:
data_type: float32
min_value: -27.803293
max_value: -27.804540
longitude:
data_type: float32
min_value: -50.336975
max_value: -50.339698
